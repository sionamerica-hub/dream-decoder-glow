## 1. 서비스 소개서 (`SERVICE_INTRODUCTION.md`)

다음 내용을 포함하는 한국어 서비스 소개 문서를 작성합니다:

### 목차
- **서비스 개요**: LucidMind의 비전과 핵심 가치 (AI 기반 꿈 분석 심리치료 플랫폼)
- **핵심 기능 소개**
  - 꿈 기록: 텍스트 입력을 통한 꿈 내용 기록
  - 일상 사건 연결: 업무/학업, 대인관계, 금전, 건강, 자아실현 등 6개 카테고리
  - XY 좌표 감정 선택: 긍정/부정(X축) x 에너지 높낮이(Y축) 2차원 감정 매핑
  - AI 심리 분석 보고서: 7개 카테고리 (핵심 메시지, 꿈 유형, 상징 분석, 감정-사건 연결, 무의식 메시지, 심리적 통찰, 행동 지침) + 위로 메시지
  - 꿈 캘린더 및 기록 관리
  - 감정 분포 통계
- **기술 스택**: React, TypeScript, Tailwind CSS, Gemini AI, Lovable Cloud
- **디자인 컨셉**: 다크 테마, 네온 그라데이션, 글래스모피즘
- **타겟 사용자**: 자기 이해와 심리적 성장에 관심 있는 사용자

## 2. 개발일지 (`DEVELOPMENT_LOG.md`)

대화 히스토리를 기반으로 시간순 개발 기록을 작성합니다:

### 기록할 개발 단계

| 단계 | 내용 |
|------|------|
| Phase 1 | 프로젝트 초기 설정 및 UI 컴포넌트 구축 (다크 테마, GlowCard, BottomNav, DreamInput, MoodSelector, EventSelector, DreamCalendar) |
| Phase 2 | AI 꿈 분석 엔진 구현 (Edge Function, Gemini 모델 연동, 융 심리학 기반 프롬프트 설계) |
| Phase 3 | 데이터베이스 스키마 설계 (dreams 테이블, RLS 정책, 인덱스) |
| Phase 4 | 사용자 인증 시스템 구현 (이메일/비밀번호 가입/로그인, ProtectedRoute) |
| Phase 5 | AI 분석 고도화 및 UX 개선 (Gemini 3 Pro 업그레이드, 7개 분석 카테고리 확장, 공감적 어조 강화, localStorage 기반 저장으로 전환, AnalysisReport UI 리뉴얼) |

각 단계별로 **변경 사항**, **기술적 결정 사항**, **사용 기술**을 기록합니다.
